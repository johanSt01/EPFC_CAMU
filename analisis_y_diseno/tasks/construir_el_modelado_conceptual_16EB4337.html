Task: Construir el modelado conceptual
Creación de un modelo de alto nivel que represente los conceptos y las relaciones clave dentro del dominio de la base de datos
Purpose
capturar los aspectos esenciales y comprender la estructura lógica de la base de datos
Relationships
RolesPrimary Performer:
  • DBA
Additional Performers:
InputsMandatory:
  • Diagramas
    		Optional:
    • None
    Outputs
    • Diagrama de entidad-relacion
      Main Description
      El DBA comienza construyendo un modelado conceptual al analizar los requisitos y las necesidades del sistema. Utilizando diagramas de entidad-relación el DBA identifica las entidades clave dentro del dominio de la base de datos y define sus atributos. También se establecen las relaciones básicas entre las entidades
