Artifact: Modelo de datos
El modelo de datos se utiliza para representar los datos dentro de una base de datos
Purpose
 Estos modelos proporcionan una estructura y un conjunto de reglas para describir cómo se almacenan, acceden y manipulan los datos.

Description
Main Description

Algunos de los modelos de datos reconocidos son:

  • Modelo Relacional: Es el modelo de datos más utilizado en la actualidad. Se basa en la teoría de conjuntos y utiliza tablas para representar entidades y relaciones entre ellas. Cada tabla tiene filas que representan registros individuales y columnas que representan los atributos de los registros. Las relaciones se establecen mediante claves primarias y foráneas.
  • Modelo de Red: Este modelo se utiliza principalmente en sistemas heredados. En lugar de utilizar tablas, el modelo de red utiliza registros y enlaces para representar las relaciones entre los datos. Los registros se organizan en estructuras de árbol o gráficas, y los enlaces definen las asociaciones entre los registros.
  • Modelo Jerárquico: Similar al modelo de red, el modelo jerárquico también utiliza registros y enlaces para organizar los datos. Sin embargo, la estructura de los datos sigue una jerarquía padre-hijo, donde un registro puede tener varios registros hijos pero solo un padre. Este modelo también es utilizado en sistemas heredados.
  • Modelo de Objetos: Este modelo combina conceptos de programación orientada a objetos con la gestión de bases de datos. Permite la representación de entidades complejas como objetos, con atributos y métodos. También admite herencia, polimorfismo y encapsulamiento.
  • Modelo de Documentos: Este modelo se basa en documentos estructurados, como archivos XML o JSON. Los datos se almacenan en forma de documentos que pueden tener estructuras flexibles y anidadas. Es ampliamente utilizado en bases de datos NoSQL, especialmente en aplicaciones web y sistemas de almacenamiento de datos semi-estructurados.
