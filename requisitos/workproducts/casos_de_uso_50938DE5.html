Outcome: Casos de uso
Son escenarios o situaciones que describen cómo los involucrados interactúan con el sistema de software. Representan funcionalidades o acciones específicas que el sistema debe ser capaz de realizar para satisfacer las necesidades de los usuarios o cumplir con los requisitos del negocio.
Purpose
Comprender y comunicar cómo se utilizará el sistema en situaciones del mundo real

Main Description
Se utilizan para capturar y especificar las interacciones entre los actores y el sistema. Estos casos describen las acciones y eventos que ocurren durante la interacción, así como las respuestas esperadas del sistema. Los casos de uso se utilizan para establecer límites y responsabilidades de los componentes del sistema
Key Considerations
Al utilizar casos de uso, se mejora la comprensión de los requisitos y se facilita la toma de decisiones de diseño orientadas a la experiencia del usuario
